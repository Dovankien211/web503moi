Luồng chi tiết: Client → App → Middlewares → Router → Handler → Response
Client
Gửi HTTP request tới API
GET /api/posts
GET /api/posts/:id
POST /api/posts
PUT /api/posts/:id
DELETE /api/posts/:id
Express App (src/app.js)
Tạo app, đăng ký middlewares, mount routers
import postRouter from "./routers/post"; import userRouter from "./routers/user"; import productRouter from "./routers/product"; app.use(morgan("dev")); app.use(express.json()); app.use("/api/posts", postRouter); app.use("/api/users", userRouter); app.use("/api/products", productRouter);
Middlewares
Theo thứ tự: morgan (log) → express.json (parse body)
app.use(morgan("dev")); app.use(express.json());
Router Matching
postRouter nhận các path con sau prefix /api/posts
postRouter.get("/", ...) postRouter.get("/:id", ...) postRouter.post("/", ...) postRouter.put("/:id", ...) postRouter.delete("/:id", ...)
Handler Logic
Thao tác mảng posts: tìm kiếm theo id, thêm, cập nhật, xóa
// Ví dụ: GET detail const id = Number(req.params.id); const post = posts.find(p => p.id === id); if (!post) return res.status(404).json({ error: "Post not found" }); return res.json(post);
Response
Trả JSON + HTTP status về Client
200 OK • GET/PUT/DELETE
201 Created • POST
400 Bad Request • thiếu field
404 Not Found • sai id
Mỗi endpoint (ví dụ minh họa code)
GET /api/posts
Trả toàn bộ danh sách
postRouter.get("/", (req, res) => { return res.json(posts); // 200 });
GET /api/posts/:id
Tìm theo id, không có → 404
postRouter.get("/:id", (req, res) => { const id = Number(req.params.id); const post = posts.find((p) => p.id === id); if (!post) return res.status(404).json({ error: "Post not found" }); return res.json(post); });
POST /api/posts
Validate body đơn giản, tạo id bằng Date.now()
postRouter.post("/", (req, res) => { const { title, content } = req.body || {}; if (!title || !content) { return res.status(400).json({ error: "Title and content are required" }); } const newPost = { id: Date.now(), title, content }; posts.push(newPost); return res.status(201).json(newPost); });
PUT /api/posts/:id
Cập nhật chọn lọc field gửi lên
postRouter.put("/:id", (req, res) => { const id = Number(req.params.id); const post = posts.find((p) => p.id === id); if (!post) return res.status(404).json({ error: "Post not found" }); const { title, content } = req.body || {}; post.title = title ?? post.title; post.content = content ?? post.content; return res.json(post); });
DELETE /api/posts/:id
Xóa theo index tìm được
postRouter.delete("/:id", (req, res) => { const id = Number(req.params.id); const index = posts.findIndex((p) => p.id === id); if (index === -1) return res.status(404).json({ error: "Post not found" }); posts.splice(index, 1); return res.json({ success: true }); });
Vai trò các file trong src
src/app.js
Điểm khởi đầu của ứng dụng Express: - Tạo app, đăng ký middlewares morgan + express.json. - Định nghĩa route gốc "/" để test nhanh. - Mount routers: /api/posts, /api/users, /api/products. - Lắng nghe cổng 3000.
src/routers/post.js
Định nghĩa nhóm endpoint cho tài nguyên Posts: - Dữ liệu giả mảng posts. - CRUD: GET list, GET detail, POST create, PUT update, DELETE remove. - Trả JSON + HTTP status phù hợp.
src/routers/user.js
Ví dụ router đơn giản cho Users: - GET /api/users → trả chuỗi "user". - GET /api/users/greet → trả chuỗi "user greet". - Minh họa cách tổ chức router độc lập.
src/routers/product.js
Ví dụ router đơn giản cho Products: - GET /api/products → trả chuỗi "product". - GET /api/products/greet → trả "product greet". - Tương tự users, cho thấy mỗi tài nguyên có router riêng.
